Step-by-Step Deployment

Panduan Lengkap Deploy ke Docker Swarm

Dari nol sampai service berjalan stabil. Ikuti urutan ini untuk environment single-manager atau cluster kecil.

Prasyarat

  • Docker Engine sudah terpasang dan aktif.
  • Port aplikasi (contoh 3000) boleh diakses dari host.
  • Project punya Dockerfile dan bisa di-build lokal.

Step 1 - Aktifkan Swarm

Jadikan Node Sebagai Manager

docker swarm init

Perintah ini membuat cluster swarm baru dan node saat ini jadi manager utama.

Cek Status Node

docker node ls

Pastikan node berstatus Ready dan role-nya Manager.

Step 2 - Build dan Push Image

Build Image

docker build -t nodejs-app:1.0 .

Image lokal siap dipakai untuk membuat service.

(Opsional) Push ke Registry

docker tag nodejs-app:1.0 your-registry/nodejs-app:1.0 docker push your-registry/nodejs-app:1.0

Wajib dilakukan kalau worker node berada di host berbeda.

Step 3 - Deploy Service ke Swarm

Buat Service

docker service create --name webapp --replicas 3 -p 3000:3000 nodejs-app:1.0

Swarm akan membuat 3 task container dan mendistribusikannya secara otomatis.

Verifikasi Service

docker service ls docker service ps webapp

Cek jumlah replika dan kondisi task apakah Running.

Step 4 - Scale dan Update

Scale Replika

docker service scale webapp=5

Naikkan replika saat trafik tinggi.

Rolling Update Image

docker service update --image nodejs-app:1.1 webapp

Update versi aplikasi tanpa menghentikan service total.

Step 5 - Hapus Service dan Keluar Swarm

Cleanup

docker service rm webapp docker swarm leave --force

Gunakan langkah ini saat lab/testing selesai agar host bersih.
